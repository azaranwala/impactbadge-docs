1. Executive Summary

ImpactBadge is a K-12 volunteer hour tracking and management platform for students aged 11–18. It enables students to discover community service opportunities, log and verify hours with GPS or QR check-in, earn gamification rewards, and export verified records for college applications. School administrators gain compliance dashboards and district reporting; volunteer organizations receive a lightweight CRM and verification workflow. Parents can monitor activity and grant COPPA consent for minors.

The platform operates across three client surfaces — an iOS mobile app (live on App Store, v1.0), an Android mobile app (code complete, pre-Play Store launch), and a React web portal (impacthub-prod.web.app) — backed by a Firebase cloud backend (130+ Cloud Functions, Firestore, Firebase Auth, Cloud Storage).

2. Product Vision & Goals

Mission

Empower every K-12 student to discover meaningful community service, build a verified volunteer record, and present that record confidently to colleges — while giving schools and organizations the administrative tools they need without complex software procurement.

Adoption Strategy

  • District-led rollout: Clever Complete integration lets district IT enable ImpactBadge with one click — zero password friction for students already in Clever.
  • Freemium: App is free for individual students; per-school pricing for institutional features (compliance reports, Clever Secure Sync, district dashboard).
  • Org network effect: Volunteer organizations post events; students discover them → more orgs join to reach student audience.
  • College application pull: Common App and Naviance export creates urgency for juniors and seniors to maintain hours year-round.
  • Teacher referral: Advisor portal gives teachers visibility; teachers recommend app to students in service-learning classes.

5.2 Security

Firestore Security Rules

  • All reads and writes require authentication; allow read, write: if false is the default deny rule.
  • Students can only read/write their own user document and hour logs.
  • Org admins can read/write events and hour logs for their organization only (verified by orgId claim).
  • School admins can read all student documents in their schoolId.
  • Teachers can read (not write) student documents in their schoolId.
  • cleverDistricts collection: deny all client access — server-only via Admin SDK.
  • Parent can read their linked student's documents only (verified by linkedStudentIds array).

COPPA Compliance

  • Users age < 13 require verifiable parental consent before any data is collected or stored beyond the consent process itself.
  • Parent consent stored with timestamp, IP address, parent UID, and consent text version.
  • Under-13 accounts do not appear in leaderboards by default.
  • No behavioral advertising targeting minors.
  • Data deletion request honored within 30 days (triggers cascade delete of all user data).

FERPA Compliance

  • Student education records (hours, grades, school info) not shared with third parties without school consent.
  • School admin data export is access-controlled to authorized school personnel only.
  • Clever-synced roster data treated as FERPA-protected; stored in restricted Firestore collections.
  • Audit log maintained for all admin reads of student data.

Vulnerability Management

  • 2 critical + 6 high + multiple medium vulnerabilities patched as of July 2026.
  • Dependabot alerts monitored; packages upgraded on security advisory.
  • CSRF protection on all OAuth callback endpoints (state parameter validation).
  • HMAC signature validation on all inbound webhooks.
  • Input sanitization and rate limiting on all public Cloud Functions.

6. Data Model

ImpactBadge uses Cloud Firestore (NoSQL document database) as its primary data store. Key collections:

Collection / PathPurposeAccess
users/{uid} Core user document. Fields: displayName, email, role, schoolId, orgId, xp, level, streakCount, cleverUserId, cleverApiVersion, coppaConsentStatus, linkedParentUid, studentId, graduationYear, skills, availability, emailVerified, fcmTokens. Owner read/write; school_admin read; parent read (if linked)
users/{uid}/hourLogs/{logId} Individual hour log entries. Fields: eventId, orgId, date, hours, status (pending/verified/rejected), verificationMethod (gps/pin/qr/self_report), gpsCoordinates, supervisorEmail, rejectionReason, autoVerified, createdAt. Owner read/write; org_admin read/update (status); school_admin read
users/{uid}/notifications/{notifId} In-app notification items. Fields: type, title, body, read, createdAt, deepLink. Owner read/write only
events/{eventId} Volunteer events. Fields: name, description, category, orgId, startTime, endTime, address, geoPoint, slots, filledSlots, shifts[], verificationMethod, pinCode, status (active/cancelled/completed). Public read; org_admin write (own org only)
events/{eventId}/registrations/{uid} Per-user registration record. Fields: userId, shiftId, registeredAt, status (registered/waitlisted/attended). Owner read; org_admin read/write
organizations/{orgId} Volunteer organization profiles. Fields: name, description, logoUrl, adminUids[], website, phone, categories[], verifiedOrg. Public read; org_admin write (own org)

users/{uid}/badges/{badgeId} Badges earned by a student. Fields: earnedAt, badgeId. Owner read; public read; admin write
challenges/{challengeId} Active challenges. Fields: title, description, xpReward, orgId, schoolId (optional scoping), startDate, endDate, criteria{}. Public read; org_admin/school_admin write
stories/{storyId} Student reflection stories. Fields: userId, eventId, text, photoUrl, createdAt, likes, featured. Public read (if user opted in); owner write
parentConsents/{consentId} COPPA consent audit records. Fields: studentUid, parentUid, consentedAt, ipAddress, consentTextVersion, status (granted/denied/revoked). Backend/admin only — no client read
cleverDistricts/{districtId} Clever district tokens and sync state. Fields: districtToken (encrypted), lastSyncAt, syncCursor, studentCount, health. Backend Admin SDK only — deny all client access
cleverSchools/{cleverSchoolId} Synced Clever school records. Fields: name, districtId, cleverSchoolId, studentCount, teacherCount, lastSyncAt. school_admin read; teacher read
cleverSections/{sectionId} Synced Clever class sections. Fields: name, teacherId, studentIds[], cleverSectionId, schoolId. school_admin read; teacher read (own sections)
pendingStudents/{pendingId} Pre-provisioned students from Clever sync awaiting first login. Fields: cleverUserId, email, name, schoolId, grade. Merged into users/ on first SSO login. Backend Admin SDK only
webhooks/{webhookId} Org webhook registrations. Fields: orgId, url, events[], secret (HMAC key), active, failureCount. org_admin write; backend read
