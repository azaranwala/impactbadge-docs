5. Clever Complete Integration

5.1 SSO Flow (Phase 1)

Client → https://clever.com/oauth/authorize?client_id=...&redirect_uri=...
Clever → callback with ?code=...
cleverOAuthCallback:
  1. Exchange code for access_token (Basic auth)
  2. GET /v3.1/me → cleverId
  3. GET /v3.1/users/{id} → roles, name, email, school, grade
  4. Match: cleverUserId → studentId+school → email → create new
  5. Merge fields (non-destructive: never blank existing data)
  6. Create Firebase custom token → redirect/return

5.2 Account Matching Priority

  1. cleverUserId — Firestore query (works for returning Clever users)
  2. studentId + schoolName — Clever Complete only (Library has no studentId)
  3. email — Firebase Auth getUserByEmail
  4. Create new — Firebase Auth createUser (no email for Library students)

5.3 Non-Destructive Merge Rules

FieldClever has valueClever is blank
schoolNameOverwrite (source of truth)Preserve existing
studentIdOverwritePreserve existing
displayNameOverwrite (if not "Student")Preserve existing
emailNever overwrite for existing usersN/A

5.4 Roster Sync (Phase 2)

Daily 2:00 AM EST — cleverRosterSyncJob:
  For each enabled district:
    1. Fetch all schools via district app token
    2. Fetch all students → upsert (match or pendingStudents)
    3. Fetch all teachers → upsert
    4. Fetch all sections → upsert
    5. Archive removed users (cleverArchived: true, never delete)

Every 15 min — cleverEventsPollJob:
  Poll /v3.1/events → process users.created/updated/deleted, sections.*

5.5 LMS Connect (Phase 3)

Grade Calculation:
  grade = min((hoursCompleted / requiredHours) × pointsPossible, pointsPossible)

Auto-push: When hours are verified → autoPushGradeIfLMSConnected()
  → finds matching LMS assignments for student's sections
  → pushes grade to Clever Resources API → Clever pushes to LMS

LTI 1.3 Launch: POST /ltiLaunchHandler
  → decode JWT id_token → extract Clever user ID
  → create/link Firebase Auth → redirect to web app with token
