2. What's Already Built

2.1 SSO (Phase 1 — Complete)

OAuth 2.0 flow via cleverOAuthCallback. Handles student, teacher, and staff roles. Creates or links Firebase Auth account, merges Clever data non-destructively, and redirects to web portal with custom token.

2.2 Secure Sync (Phase 2 — Complete)

Daily Full Sync — cleverRosterSyncJob (Pub/Sub, 2AM EST)
Iterates all enabled districts → schools → students → teachers → sections → archive removed users.

Delta Sync — cleverEventsPollJob (every 15 min)
Polls /events?starting_after={lastEventId}. Processes: users.created, users.updated, users.deleted, sections.created, sections.updated, sections.deleted.

Admin Setup — connectCleverDistrictCall
School admin enters district ID + app token. Token validated by calling /districts/{id} before saving.

Manual Sync — triggerCleverSyncCall (rate limited 1/hr)
School admin triggers immediate sync from web portal Clever Sync page.

2.4 Matching Strategy

For students:
  1. Query users where cleverUserId == student.id → update
  2. If not found: query users where studentId == sis_id AND schoolName == school → update
  3. If not found: query users where email == student.email → update
  4. If not found: write to pendingStudents/{email|clever_id} → merge when student signs up

For teachers:
  1. Query users where cleverUserId == teacher.id → update
  2. If not found: Firebase Auth getUserByEmail → update existing user doc
  3. If not found: createUser in Firebase Auth + new user doc

2.5 LMS Connect (Phase 3 — Complete)

Grade passback via Clever Resources API. Volunteer hours logged → grade calculated (hours / required × points). LTI 1.3 launch supported. Auto-pushes grades when hours are verified.

Now that rostering scopes are active, these features can be built that were not possible with Clever Library alone:
