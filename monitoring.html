ConditionMore than 5 server errors (status = "error") in a 5-minute window
SeverityCRITICAL
What It CatchesUnhandled exceptions, Firestore failures, crashed functions, timeout errors
Auto-Close30 minutes after condition clears
ActionCheck Cloud Functions logs → filter by severity=ERROR → identify failing function → fix and redeploy

2.2 Cloud Functions — High 4xx Error Rate

ConditionMore than 20 client errors (non-ok, non-error status) in a 10-minute window
SeverityWARNING
What It CatchesBad input patterns from a broken app update, validation failures, potential abuse/attacks
Auto-Close30 minutes
ActionCheck which function is receiving 4xx → review input validation → check if a new app version has a bug

SeverityWARNING

ConditionAny log entry containing "requires an index" at ERROR severity
SeverityCRITICAL
What It CatchesMissing composite Firestore index — completely blocks the affected feature
Rate LimitMax 1 notification per hour
ActionClick the index creation link in the log entry → wait 1-3 min for index to build → add index to firestore.indexes.json → deploy

9. Incident Runbook

When You Get a 5xx Error Alert

  1. Go to Cloud Functions Logs → filter: severity>=ERROR
  2. Identify the failing function name and error message
  3. Check Error Reporting for error group count and first/last occurrence
  4. If "requires an index" → click the link in the log to create the index
  5. If code bug → fix locally, run tests, deploy: firebase deploy --only functions
  6. Verify the fix in logs (status should change from 500 to 200)

When You Get an Uptime Alert

  1. Go to Uptime Checks → check which endpoint is down
  2. If web portal → check Firebase Hosting status
  3. If Cloud Function → check deployment status in Firebase Console
  4. Try calling the endpoint manually: curl https://us-central1-impacthub-prod.cloudfunctions.net/FUNCTION_NAME
  5. If all endpoints are down → check Google Cloud Status Dashboard for regional outage

When You Get a Billing Alert

  1. Go to Billing Budgets → check which services are driving cost
  2. Check Firestore Document Reads on the dashboard — this is usually the biggest cost driver
  3. If Firestore reads are spiking → check for missing query limits or runaway listeners
  4. If Cloud Functions are spiking → check for infinite loops or retry storms
  5. At $500 → consider enabling budget-based Cloud Function disabling
